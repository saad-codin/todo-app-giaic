# Implementation Plan: Premium UI/UX Redesign

**Branch**: `007-premium-ui-redesign` | **Date**: 2026-02-22 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/007-premium-ui-redesign/spec.md`

---

## Summary

Complete visual redesign of the Next.js 14 frontend to a premium, polished productivity app experience. The approach installs shadcn/ui, framer-motion, next-themes, and lucide-react, then replaces the visual layer of all pages and components while **preserving every existing hook, API call, and business logic unchanged**. The result is a sidebar-navigation layout with sage green accents, dark mode, smooth animations, a marketing-grade landing page, and polished auth screens.

---

## Implementation Phases

### Phase A — Foundation (Dependencies + Design System)

**Goal**: Install all new libraries, configure Tailwind + shadcn for dark mode, establish the CSS variable palette.

**Steps**:

1. **Install packages**
   ```bash
   cd frontend
   npm install framer-motion next-themes lucide-react
   npx shadcn@latest init   # style=Default, base=Neutral, CSS vars=Yes
   npx shadcn@latest add button card badge input dialog dropdown-menu sheet separator scroll-area tooltip label textarea
   ```

2. **Update `tailwind.config.js`**
   - Ensure `darkMode: ["class"]`
   - Ensure `content` includes shadcn paths
   - Add sage green color token to extend colors:
     ```js
     colors: {
       sage: { 50: '#f2f7f4', 100: '#e0ede6', 200: '#c2dace', 300: '#97c1a9', 400: '#66a080', 500: '#3d8b6e', 600: '#2e6d54', 700: '#245642', 800: '#1c4333', 900: '#142e24' }
     }
     ```

3. **Update `globals.css`**
   - shadcn init writes CSS variables; customize `--primary` to sage green HSL
   - Light: `--primary: 152 40% 40%;`
   - Dark: `--primary: 152 40% 55%;`

4. **Update `app/layout.tsx`**
   - Import and wrap with `ThemeProvider` from next-themes
   - Add `suppressHydrationWarning` to `<html>`
   ```tsx
   import { ThemeProvider } from 'next-themes';
   <html lang="en" suppressHydrationWarning>
     <body>
       <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
         <Providers>{children}</Providers>
       </ThemeProvider>
     </body>
   </html>
   ```

5. **Create `ThemeToggle.tsx`**
   - Uses `useTheme()` from next-themes
   - Sun/Moon icons from lucide-react
   - Smooth icon swap animation with framer-motion

### Phase B — Sidebar & Layout Shell

**Goal**: Replace the existing sidebar and dashboard layout with the new sidebar navigation structure. Mobile overlay via `Sheet` component.

**Sidebar new structure**:
```
Logo + brand name
─────────────────
Navigation
  • Dashboard (LayoutDashboard icon)
  • AI Assistant (Bot icon)
  • Calendar (Calendar icon)
  • Sticky Wall (StickyNote icon, placeholder route)
─────────────────
My Lists
  • Personal (tag with dot)
  • Work (tag with dot)
─────────────────
Tags
  [tag pills from user's tags]
─────────────────
[Theme toggle]
[User avatar + name + sign out]
```

**Key implementation details**:
- Desktop: fixed 264px sidebar, `w-64` with `lg:flex` — never collapses
- Mobile: hidden (`hidden lg:flex`), triggered via `Sheet` component (slide-in overlay)
- Active item: `bg-sage-100 text-sage-700` with left border accent `border-l-2 border-sage-500`
- Hover: `hover:bg-gray-50 dark:hover:bg-gray-800` with `transition-colors duration-150`
- framer-motion: `whileHover={{ x: 2 }}` on nav items for subtle slide effect

**`dashboard/layout.tsx`** updates:
- Keep auth guard logic (`useAuthContext`, redirect to `/signin`)
- Replace inline sidebar with new `<Sidebar>` component
- Replace header with new `<Header>` (mobile only)
- Add `MobileSidebarTrigger` prop/state for Sheet open/close

### Phase C — Landing Page

**Goal**: Marketing-grade landing page with hero, features, testimonials area, and CTA. Scroll-triggered animations with framer-motion.

**Sections**:
1. **Hero** — Full-width, headline "Organize your work, amplify your focus", subheadline, two CTAs (Get Started → /signup, Sign In → /signin). Subtle background gradient (sage-50 to white).
2. **Features Grid** — 3 or 6 feature cards with icons: Task Management, AI Assistant, Calendar View, Dark Mode, Real-time Sync, Recurring Tasks.
3. **CTA Banner** — "Start for free today" with prominent button.
4. **Footer** — Simple footer with brand name and links.

**Animations**:
- Hero text fades up on mount (`initial={{ opacity: 0, y: 20 }}`)
- Feature cards stagger in on scroll (use `whileInView` + `viewport={{ once: true }}`)
- CTA button has `whileHover={{ scale: 1.03 }}` pulse effect

### Phase D — Auth Pages

**Goal**: Beautiful sign-in and sign-up pages. Split-screen or centered card layout.

**Layout**: Centered card on a soft gradient background (`bg-gradient-to-br from-sage-50 to-gray-100`).
- Card: `bg-white dark:bg-gray-900 shadow-xl rounded-2xl p-8 w-full max-w-md`
- Logo + app name above form
- Form uses shadcn `Input`, `Label`, `Button` components
- Error messages: inline below field with `text-red-500 text-sm` and slide-down animation
- "Already have an account? Sign in" link at bottom

**Animation**: Card entrance with `initial={{ opacity: 0, y: 30 }} animate={{ opacity: 1, y: 0 }}`.

### Phase E — Dashboard Page & Task Components

**Goal**: Redesign the main dashboard visuals while keeping all existing logic hooks intact.

**Dashboard layout**:
```
┌─────────────────────────────────────────────────────┐
│ Header: "Good morning, {name}" | Search | + New Task│
├─────────────────────────────────────────────────────┤
│ Progress bar (completed/total tasks today)          │
├─────────────────────────────────────────────────────┤
│ Filter tabs: All | Urgent | High | Medium | Low     │
│ [search input on right]                             │
├─────────────────────────────────────────────────────┤
│ Task list (animated, staggered)                     │
│  ┌─────────────────────────────────────────────────┐│
│  │ ○ Task description              [High] [work] ││
│  │   📅 Mar 15 · 🔁 Weekly         [Edit][Delete] ││
│  └─────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘
```

**TaskCard redesign**:
- Card: `bg-white dark:bg-gray-800 rounded-xl border border-gray-100 dark:border-gray-700 shadow-sm`
- Hover: `shadow-md` transition + `motion.div whileHover={{ y: -1 }}`
- Checkbox: animated `CheckCircle2` icon (empty → green fill on complete)
- Completion: `motion.div animate={{ opacity: completed ? 0.5 : 1 }}`; description gets strikethrough
- Priority badge: shadcn `Badge` with color variants
- Action buttons appear on hover with `opacity-0 group-hover:opacity-100 transition-opacity`

**Task creation modal**:
- Uses shadcn `Dialog` component (replaces custom modal)
- Entrance animation: `initial={{ scale: 0.95, opacity: 0 }} animate={{ scale: 1, opacity: 1 }}`
- All existing `TaskForm` fields preserved: description, priority, due_date, due_time, tags, recurrence
- Focus trap and escape-to-close handled by Dialog primitives

**Empty state**:
- Centered illustration (SVG clipboard or checkmark) + "No tasks yet" message
- "Create your first task" button

**Filter tabs**:
- shadcn-styled tabs with smooth indicator slide (framer-motion `LayoutGroup`)

### Phase F — AI Assistant Page

**Goal**: Premium chat UI at `/chat` route.

**Layout**:
- Full height minus sidebar: `flex flex-col h-full`
- Message history: `flex-1 overflow-y-auto` with messages appearing from bottom
- Input area: fixed at bottom with `bg-white dark:bg-gray-900 border-t`

**Message bubbles**:
- User messages: right-aligned, `bg-sage-500 text-white rounded-2xl rounded-tr-sm`
- AI messages: left-aligned, `bg-gray-100 dark:bg-gray-800 rounded-2xl rounded-tl-sm`
- Message appearance: `initial={{ opacity: 0, y: 10 }} animate={{ opacity: 1, y: 0 }}`

**Typing indicator**: Three bouncing dots using framer-motion `animate={{ y: [0, -4, 0] }} transition={{ repeat: Infinity, duration: 0.6, stagger: 0.1 }}`

**Auto-scroll**: `useEffect` with `scrollIntoView()` on new message.

### Phase G — Dark Mode Polish

**Goal**: Verify every component looks correct in dark mode. All `dark:` variants in place.

**Checklist**:
- Sidebar: `dark:bg-gray-900 dark:border-gray-800`
- Cards: `dark:bg-gray-800 dark:border-gray-700`
- Inputs: `dark:bg-gray-700 dark:border-gray-600 dark:text-white`
- Landing page: gradient inverts (`dark:from-gray-900 dark:to-gray-800`)
- Auth pages: card `dark:bg-gray-900`
- Badges: all priority colors have dark-safe text contrast
- Modal backdrop: `dark:bg-black/60`

**WCAG AA check (SC-007)**: All text/background combinations must meet 4.5:1 contrast ratio.

---

### Phase H — Real-Time & Notification Integration

**Goal**: Integrate notification bell and WebSocket status into new sidebar/header design.

**Notification bell**: Already implemented (`NotificationBell.tsx`). Style with new design tokens. Place in sidebar footer near theme toggle.

**WebSocket status indicator**: Small dot in sidebar footer:
- Green `bg-green-400` = connected
- Gray `bg-gray-400` = disconnected
- Pulsing animation when connected (framer-motion `animate={{ scale: [1, 1.2, 1] }} transition={{ repeat: Infinity, duration: 2 }}`)
